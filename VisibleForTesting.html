@VisibleForTesting

No need to compromise visibility due to testability!

Overview

This annotation allows you to widen the access scope of a method solely for the purposes of testing. The compiler will complain if any portion of the code accesses this method if the class name containing it does not contain "Test". 

package main.myClasses;
public class MyClass {

   @VisibleForTesting
   public int add(int a, int b) {
      return a+b;
   }

}


package main.myOtherClasses;
public class MyOtherClass
{
   public MyOtherClasses() {
      // ERROR: method add is not visible
      new MyClass().add(1, 1);
   }
}


package main.tests.myOtherClasses;
public class MyOtherClassTest
{
   public MyOtherClassesTest() {
      // OK, it is visible
      new MyClass().add(1, 1); 
   }
}
